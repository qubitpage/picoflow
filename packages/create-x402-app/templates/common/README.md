# __APP_NAME__

A working **x402 paid-API** scaffold on Circle Arc Testnet.
Generated by [`create-x402-app`](https://www.npmjs.com/package/create-x402-app) (template: `__TEMPLATE__`).

## What you get

- **`src/server.ts`** — Express server that returns `402 Payment Required` for one paid endpoint, then accepts a signed EIP-3009 `transferWithAuthorization` and serves the response.
- **`src/buyer.ts`** — Buyer-agent demo: discovers the price, signs the authorization, retries with `X-PAYMENT`, prints the result.

The whole loop runs against **Arc Testnet** (chainId `5042002`, USDC at `0x36...0000`). No mainnet funds required.

## Quick start

```bash
npm install
cp .env.example .env
# put any test EOA private key into BUYER_PRIVATE_KEY (you can generate one with viem.generatePrivateKey())

# terminal 1 — paid API
npm run dev:server

# terminal 2 — buyer agent
npm run demo
```

You should see `402 Payment Required → signed → 200 OK` in under a second.

## What's in the box

- [`@picoflow/x402-facilitator`](https://www.npmjs.com/package/@picoflow/x402-facilitator) — challenge builder, EIP-712 verifier, JSON Schema, optional `/price_quote` engine.
- [`@picoflow/agent-wallet`](https://www.npmjs.com/package/@picoflow/agent-wallet) — `paidCall(url)` that handles 402 transparently.

## Where to go next

- **Add more endpoints** — wrap any handler with the `paid()` middleware in `src/server.ts`.
- **Negotiate a price** — call `wallet.requestQuote(url, { volume: 1000 })` to get a discount before signing.
- **Settle on-chain** — set `RELAYER_PRIVATE_KEY` on the server and add a worker that drains the outbox; see [PicoFlow GatewayWorker](https://picoflow.qubitpage.com/docs).
